# 项目：德州哲学家（Texas Philosopher）
# 类型：基于频域决策哲学的AI驱动德州扑克训练游戏
# 目标：将"低成本试错、鱼塘密度探测、独立事件思维"转化为可交互的游戏机制
# 开发框架：Python + 终端TUI（Text User Interface）或Web（优先TUI可快速验证）

## 一、核心哲学锚定（必须严格嵌入游戏机制）

游戏必须训练玩家以下思维模式，所有机制设计必须服务于这些原则：

1. **独立事件原则**：每一局牌都是独立事件，上一局的输赢、情绪、手牌质量不得影响下一局决策。游戏必须设计"心理账户污染"惩罚机制。

2. **成本管理优先**：玩家的核心资源不是筹码，而是"时间池"和"资金池"。游戏必须让玩家体验"pool不被耗干"比"赢一把大的"更重要。

3. **鱼塘密度探测**：玩家必须学会用极低成本快速判断一个牌桌是否值得继续（对手水平、牌桌动态）。不值得立刻换桌（开下一局）。

4. **捡钱而非挣钱**：只打正期望值（+EV）的牌局。游戏必须让玩家感受到"勉强参与"的惩罚远大于"错过机会"的代价。

5. **快速试错与迭代**：鼓励玩家在短时间内开多局（低成本探测），而非在一局上过度投入。

## 二、游戏核心架构

### 2.1 游戏模式："探宝航海"模式（大航海式探宝）

不要做成传统德州扑克的"坐下一桌打到底"。而是做成**地图探索式**（参考杀戮尖塔爬塔）：

- 玩家是一艘"探宝船"的船长，拥有**总资金池**（Pool）和**时间池**（Time Pool）。
- 地图上分布着不同的"鱼塘"（牌桌），每个鱼塘有标签：
  - 买入成本（Buy-in）
  - 盲注结构（Blinds）
  - 预估密度（Density Rating：鱼/鲨鱼比例）
  - 特殊规则（Mutation，参考小丑牌）
  - 预估时长（Time Cost）
- 玩家选择进入哪个鱼塘，打完一局后可以选择：继续（追加投入）或离开（换鱼塘/回港口）。

### 2.2 双资源系统（核心创新）

- **资金池（Pool）**：总资金，不能归零。归零=游戏结束（Game Over）。
- **时间池（Time Pool）**：每局消耗时间，每天恢复一定时间。时间=生命。
- **声望值（Reputation）**：连胜、正确决策积累，解锁高级鱼塘。
- **经验值（XP）**：用于解锁"思维技能树"。

### 2.3 小丑牌（Balatro）式突变机制

每个鱼塘除了标准德州规则，还带有1-3个"突变"（Mutations），强迫玩家在极限状态下思考：

示例突变池：
- **"沉没成本"诅咒**：弃牌后，下一局如果参与必须强制多投入50%（训练玩家克服心理账户）。
- **"梭哈狂热"**：每局强制至少有一名玩家All-in（训练玩家在极端压力下的风险管理）。
- **"信息黑洞"**：看不到自己的一张底牌（训练概率思维和信息不完整决策）。
- **"密度迷雾"**：进入鱼塘前看不到对手等级，必须打1手才能探测（低成本试错训练）。
- **"时间加速"**：每手牌思考时间限制10秒（训练快速决策）。
- **"独立事件封印"**：显示上一局结果干扰本局（训练独立事件思维，克服赌徒谬误）。
- **"捡钱法则"**：只有拿到AA/KK/AK才允许参与，否则强制弃牌（训练严格牌力筛选）。

### 2.4 杀戮尖塔（Slay the Spire）式路线选择

- 地图为节点网络，每个节点是一个"鱼塘"或"事件"。
- 路线选择：玩家可以选择"安全路线"（低买入、低密度、低收益）或"危险路线"（高买入、高密度未知、高收益）。
- 港口事件：在节点之间可以触发随机事件（参考杀戮尖塔的"？"房间）：
  - "遇到老渔夫"：获得一个鱼塘的密度情报
  - "风暴来临"：时间池减少，但下一局收益翻倍
  - "发现新海域"：解锁一个隐藏鱼塘
  - "修船"：消耗资金恢复时间池（训练资源转换思维）

### 2.5 思维技能树（参考杀戮尖塔卡牌构建）

玩家通过XP解锁"思维技能"（被动能力），这些能力改变游戏玩法：

- **"快速弃牌"**：弃牌动作不消耗时间池（训练及时止损）。
- **"鱼塘雷达"**：进入前可看到对手AI的等级标签（训练信息收集）。
- **"情绪隔离"**：连续输3局后，第4局资金损失减少50%（训练情绪管理）。
- **"乌龙指嗅觉"**：每局有5%概率发现"捡钱机会"（直接获得小额奖励，无需打牌）。
- **"独立事件护盾"**：上一局结果不会显示在本局（强制独立事件思维）。
- **"低成本探测"**：第一局买入成本减半（鼓励快速试错）。

## 三、AI对手系统（核心要求）

### 3.1 每个AI对手必须是一个独立Agent

每个对手必须有：
- **姓名与背景故事**：与文章哲学相关的身份（如"老渔夫"、"梭哈狂人"、"学院派"、"捡钱者"）。
- **可见性格标签**：如"激进"、"保守"、"情绪化"、"数学派"。
- **决策逻辑**：基于概率、筹码深度、位置、对手历史的混合模型。
- **语言交流能力**：在关键决策点（玩家行动后、亮牌后、all-in时）会主动对玩家说话。

### 3.2 AI对手对话示例（必须实现）

AI对手在以下时机必须生成对话：
- **玩家All-in时**："你确定？这不像你的风格，是不是上局输急了？"（测试玩家情绪稳定性）
- **玩家弃牌后**："又弃了？这鱼塘密度这么大，你都不敢下网？"（测试玩家是否过度保守）
- **玩家连续弃牌5手后**："你只负责捡钱，不负责挣钱，我理解。但鱼都游走了。"（引用文章哲学）
- **AI自己All-in时**："我pool还深，死不了就搞。你跟不跟？"（展示非对称风险思维）
- **玩家赢了大锅后**："独立事件，下一局归零。别把你的运气当实力。"

### 3.3 AI对手类型（至少6种）

1. **"老渔夫"（The Fisherman）**：
   - 策略：只打 premium hands，极度耐心，高密度才出手。
   - 台词："我等了3小时就为这一手，鱼塘密度不够我宁可晒太阳。"
   - 训练价值：让玩家看到"捡钱"的典范。

2. **"梭哈狂人"（The Gambler）**：
   - 策略：高频All-in，情绪驱动，模仿文章中的"梭哈"反面教材。
   - 台词："人生就是梭哈！这一把我要翻本！"
   - 训练价值：让玩家看到情绪失控的代价。

3. **"密度探测者"（The Scout）**：
   - 策略：前几手极小成本试探，探测后立即调整策略。
   - 台词："我先扔个饵，看看这池子里是鱼还是鲨鱼。"
   - 训练价值：展示低成本试错。

4. **"学院派"（The Academic）**：
   - 策略：严格GTO（Game Theory Optimal），概率精确，但缺乏灵活性。
   - 台词："根据ICM模型，你的跟注概率应该只有12.7%。"
   - 训练价值：让玩家理解过度优化的局限。

5. **"后浪"（The Young Wave）**：
   - 策略：激进、学习快、模仿玩家策略。
   - 台词："你这套过时了，新的打法是不断换桌，找到最软的鱼塘。"
   - 训练价值：提醒玩家没有永远的舒适区。

6. **"虚空建筑师"（The Void Architect）**：
   - 策略：频域思维，不预测单局，只统计频率，在特定频率点出击。
   - 台词："我不看这一手牌，我看这一百手的频谱。现在，振幅到了。"
   - 训练价值：展示文章核心哲学。

## 四、多邻国（Duolingo）式每日激励系统

### 4.1 每日机制

- **每日时间池**：每天恢复固定时间（如30分钟），不累积（训练每日持续思考）。
- **连胜（Streak）**：连续登录天数，中断归零（损失厌恶驱动）。
- **每日挑战（Daily Quest）**：
  - "今天弃掉10手烂牌"（训练止损）
  - "探测3个新鱼塘"（训练试错）
  - "在密度迷雾鱼塘赢1局"（训练不确定性承受）
  - "连续5局不受上一局结果影响"（训练独立事件）
- **经验值与等级**：从"鱼苗"→"见习渔民"→"老渔夫"→"探宝船长"→"虚空建筑师"。

### 4.2 即时反馈（参考多邻国）

- **决策评分**：每手牌结束后，系统对玩家决策给出评分：
  - "+5 XP：完美弃牌！你避免了负期望值陷阱。"
  - "+10 XP：独立事件思维！上一局输光没有影响你的这一局判断。"
  - "-5 XP：沉没成本警告！你因为上一局赢了而过度自信。"
  - "+20 XP：捡钱成功！你只在正期望值时参与并获利。"

### 4.3 排行榜与社交

- **全球/好友排行榜**：按"思维评分"（而非资金量）排名，防止单纯比钱多。
- **每周锦标赛**：特殊规则鱼塘（如"只能弃牌"挑战、"强制All-in"挑战）。

### 6.2 决策评分算法（伪代码）

```python
def score_decision(player_action, hand_strength, pot_odds, previous_result, emotional_state):
    score = 0
    feedback = []

    # 独立事件检查
    if previous_result == "lost_big" and player_action == "fold":
        score += 10
        feedback.append("独立事件思维！上一局输光没有影响你的判断。")
    elif previous_result == "won_big" and player_action == "raise":
        score -= 5
        feedback.append("沉没成本警告！上一局赢了让你过度自信。")

    # 期望值检查
    if hand_strength < 0.4 and pot_odds < 0.3 and player_action == "fold":
        score += 5
        feedback.append("完美弃牌！你避免了负期望值陷阱。")
    elif hand_strength < 0.3 and player_action == "call":
        score -= 10
        feedback.append("-EV警告！你在为别人的期望值买单。")

    # 捡钱检查
    if hand_strength > 0.8 and player_action == "raise":
        score += 5
        feedback.append("捡钱模式！好牌就要敢于价值下注。")

    return score, feedback
```

```python
class Mutation:
    def __init__(self, name, description, apply_to_round):
        self.name = name
        self.description = description
        self.apply_to_round = apply_to_round  # pre_deal, pre_flop, post_flop, etc.

    def apply(self, game_state):
        # 修改游戏状态
        pass

# 示例：沉没成本诅咒
class SunkCostCurse(Mutation):
    def apply(self, game_state):
        if game_state.last_action == "fold":
            game_state.min_bet *= 1.5
            game_state.add_notification("沉没成本诅咒激活：本局强制多投入50%")
```
