德州哲学家 v2 — 完全重建方案

     Context

     现有 v1 是一个 3,277 行的 MVP，核心问题：
     1. 不好玩：牌面动画是旋转的CSS keyframe（用户原话"所有牌转半天，很难受"）
     2. 不真实：6个AI对手只有参数差异，无人格深度
     3. 不精致：深海主题抽象数据可视化风格，不像棋牌游戏
     4. 启动bat杀进程：start http://localhost:8765 之类命令会关闭其他PowerShell
     5. 训练效果差：即时Toast反馈不是有效的概率训练方式

     用户目标：一款惊艳众人的、让人上瘾的、真实感的德州扑克哲学训练模拟器。对标 Duolingo 的日活粘性 + Balatro 的视觉冲击
      + 杀戮尖塔的爬塔成长 + PokerSnowie 的专业训练。

     核心架构原则（借鉴核岛智询）

     1. 核心-增强双层：无LLM时规则引擎必须完整可玩，LLM是增强层
     2. 降级链：DeepSeek V4 → 规则决策 + 预设台词 → 纯参数AI
     3. 哨兵协议：LLM输出用JSON schema约束，不靠正则剥离
     4. 意图-执行-验证：每个Phase交付可运行验证版本

     Phase 1：核心扑克引擎 (2天)

     目标：完整、正确的德州扑克引擎，丝滑的Canvas牌桌渲染

     关键重新设计：

     1. 状态机驱动（非v1的procedural flow）
     IDLE → DEALING → PREFLOP → FLOP → TURN → RIVER → SHOWDOWN → RESULT
     每个状态是显式的，转换可动画化

     2. Canvas牌桌（非v1的DOM卡片）
     - 椭圆绿色牌桌，金色镶边
     - 牌从中央牌堆滑向座位（不是旋转！）
     - 交错发牌动画（每张间隔120ms）
     - 翻牌：牌从背面翻到正面（Y轴旋转180°，仅0.3秒）
     - 筹码滑入底池动画
     - 所有动画用 requestAnimationFrame，GPU加速transform

     3. 边池管理（v1缺失）
     - pot-manager.ts：正确计算多个All-in场景的边池
     - 支持底池分裂

     4. EV计算器（全新核心）
     - ev-calc.ts：Monte Carlo模拟（默认300次，Web Worker可选）
     - 对每个可能的行动（fold/call/各尺度raise）计算EV（BB单位）
     - 预计算常见preflop equity（查表）
     - 输出：{ fold, call, raiseSizes, optimal, equity, potOdds }

     5. 三种模式
     - 学习模式：真实节奏 + 全辅助 + EV即时揭示
     - 训练模式：快速节奏 + 延迟辅助 + 会话报告
     - 竞技模式：真实节奏 + 零辅助 + 排名积分

     Phase 3：训练反馈系统 (1.5天)

     目标：隐藏-后-揭示式EV反馈，让玩家从"直觉决策"到"概率内化"

     核心流程：

     1. 决策前：只看到牌面、位置、对手标签。无数字。
     2. 决策后：揭示EV级联（Balatro式管道动画）：
     [手牌胜率 67%] → [底池赔率 33%] → [你的选择 CALL] → [最优 FOLD] → [差距 -2.3 BB]
     3. 每步100ms交错动画。绿色=正确，黄色=可接受，红色=偏差大
     4. 一手结束后：紧凑摘要
       - 每条街的决策对比
       - 净BB得失 vs 最优打法
       - 一句话要点："你在CO位用K5o跟注3-bet，平均损失1.8BB"
     4. 一圈结束后：报告卡
       - 漏洞检测："你在BB位过度弃牌" / "面对3-bet跟注范围过宽"
       - 会话EV：总BB盈亏 vs 最优
       - 改进区：2-3个特定场景建议
       - 闪光点：你做好的事
     5. 渐进辅助缩减：
       - 等级1-3：每次行动后完整EV揭示
       - 等级4-6：EV揭示延迟2秒，需点击查看
       - 等级7-9：仅显示正确/错误，无EV数字
       - 等级10+：仅会话报告，牌局中无反馈
